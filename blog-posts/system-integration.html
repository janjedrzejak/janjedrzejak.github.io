INTEGRATION / ARCHITECTURE

Integration is an ownership problem before it is an API problem

12 JUN 20269 MINJAN JĘDRZEJAK

Teams often describe integration as a technical connection.

An endpoint cannot resolve semantic conflict

Two systems can exchange a field named status.

Every handoff needs an owner

Who corrects an invalid record?

Architecture question

Do not ask only whether systems can connect.

Observability is part of the product

A durable integration exposes volume, latency and failures.

The API is the final expression

Once definitions and ownership are clear, implementation becomes easier.
